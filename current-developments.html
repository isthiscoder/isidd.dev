Current Developments

Last updated — June 2025

Updates

Bitslyf is now the umbrella under which all four systems — Alphaneurals, Viastora, Metauxo, and Aretenex — are being designed and documented. The portfolio at isidd.dev has moved from placeholder content to reflecting the real architecture of this work.

Building

Alphaneurals — the context and memory layer. Current focus is the retrieval pipeline: chunking strategy, hybrid dense and sparse search, and a reranking step. The open problem right now is recall on multi-hop queries, where single-pass retrieval misses information split across documents.

Learnings

Spent this cycle on attention mechanics — specifically why the scaling factor in scaled dot-product attention exists. It is not an arbitrary normalization: without it, dot product variance grows with dimension, pushing softmax into a saturated regime and killing gradients. The fix is structural, not cosmetic.

Key Insight

Retrieval quality is the ceiling on generation quality. A strong model given weak context produces confident, wrong answers — and from the outside, that failure looks like a model problem. Most "model problems" in production systems are retrieval problems wearing a model's name.

Recently Shipped

First end-to-end retrieval and generation pipeline for Alphaneurals, running against a local document set. p99 latency under 140ms at the current corpus size — the baseline this build will be measured against going forward.

Next

Build an evaluation set for the retrieval pipeline — a fixed list of queries with known-correct sources, so future changes to chunking or reranking can be measured against a baseline rather than judged by feel.

Current Thought

The gap between a model that works in a notebook and a system that works in production is mostly retrieval, latency, and evaluation — not the model itself. That gap is what this work is focused on closing, one layer at a time.
